# VERSION CONTROL
Version control is a system used to manage and track changes made to files over time. It allows you to save different versions of your work so that you can always go back to an earlier version if needed.

Instead of losing previous work when you make edits, version control keeps a record of every change, including what was changed and when it was done. This makes it easier to correct mistakes and understand how a project has developed.

It is commonly used with tools like Git and platforms such as GitHub, especially in programming, but it can also be applied to documents and other files.

version control helps you stay organized, avoid losing work, and keep track of progress in any project.


# DIFFERENCE BETWEEN GIT AND GITHUB
 Git

Git is a tool/software you install on your computer.
It is used to:

* Track changes in your files
* Save different versions of your work
* Manage your project locally

 GitHub

GitHub is an online platform/website that uses Git.
It is used to:

* Store your projects online
* Share your code with others
* Collaborate with teams

# DIFFERNCE BETWEEN GIT FETCH AND GIT PULL

Both commands are used in Git to get updates from a remote repository, but they work differently:

Git Fetch
This Downloads changes from the remote repository
It does NOT change your current files
It Lets you review updates before applying them


Git Pull
It Downloads changes from the remote repository
it Automatically applies (merges) them into your current project

Fetch is safer because it doesn’t change your work immediately
Pull is faster because it updates your project instantly
